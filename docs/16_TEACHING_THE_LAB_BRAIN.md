# 16 - Teaching the Lab Mind

This system can become a living demonstration of:

- local servers
- private networks
- AI model limits
- digital fabrication workflows
- documentation as infrastructure
- recoverable systems
- ethical technical design

## Student-safe demonstrations

- what a server is, using the cockpit page
- what a container is, using a harmless static service
- what local AI is, using the assistant with non-private prompts
- what a dashboard is, using printer or CNC status cards
- what a network boundary is, comparing LAN, tailnet, and cloud
- why we write docs, by rebuilding a service from instructions

## Avoid demonstrating

- secrets management with real secrets
- raw student data ingestion
- remote CNC control
- unrestricted network scanning
- admin panels as spectacle

## Good prompt exercise

Ask students to compare:

1. vague prompt
2. specific prompt with context
3. prompt that includes a checklist
4. prompt that asks the model to state uncertainty

Then talk about why tools need boundaries, not just power.

## Educator posture

Keep the browser-first path calm and explicit.
If the room breaks, the docs should make the rebuild boring.
