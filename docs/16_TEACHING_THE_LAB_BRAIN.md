# 16 — Teaching the Lab-Brain

## Why this is teachable

This system can become a living demonstration of:

- local servers
- private networks
- AI model limits
- digital fabrication workflows
- documentation as infrastructure
- recoverable systems
- ethical technical design

## Student-safe demonstrations

- “What is a server?” using the cockpit page
- “What is a container?” using a harmless static service
- “What is local AI?” using the assistant with non-private prompts
- “What is a dashboard?” using printer/CNC status cards
- “What is a network boundary?” comparing LAN, tailnet, and cloud
- “Why do we write docs?” rebuilding a service from instructions

## Avoid demonstrating

- secrets management with real secrets
- raw student data ingestion
- remote CNC control
- unrestricted network scanning
- admin panels as spectacle

## Good prompt exercise

Ask students to compare:

1. vague prompt
2. specific prompt with context
3. prompt that includes a checklist
4. prompt that asks the model to state uncertainty

Then talk about why tools need boundaries, not just power.
